## 1. 沙箱接口

```tsx
// src/lib/sandbox/types.ts
export interface SandboxStartOpts {
	projectId: string
	workspaceDir: string
	command: string                 // 通常是 'pnpm dev' / 'pnpm start'
	prefer?: 'child' | 'docker'     // 默认 'child'
}
export interface SandboxHandle {
	projectId: string
	port: number
	url: string
	pid?: number
	containerId?: string
	startedAt: number
	stop(): Promise<void>
}
```

## 2. 端口池 `src/lib/sandbox/port-pool.ts`

固定区间 3010 ～ 3099，进程内分配，stop 后回收。

```tsx
const RANGE = [3010, 3099] as const
const used = new Set<number>()
export function acquirePort(): number {
	for (let p = RANGE[0]; p <= RANGE[1]; p++) {
		if (!used.has(p)) { used.add(p); return p }
	}
	throw new Error('no free port')
}
export function releasePort(p: number) { used.delete(p) }
```

端口冲突探测：起完后做一次 `net.connect` ping，5s 内连不上视为失败并 stop。

## 5. 入口聚合 `src/lib/sandbox/index.ts`

```tsx
import { startChild, getRunning } from './child-process'
import { startDocker } from './docker'

export async function startSandbox(opts: SandboxStartOpts) {
	const exists = getRunning(opts.projectId)
	if (exists) return exists
	if (opts.prefer === 'docker' && process.env.SANDBOX_DOCKER === '1') {
		try { return await startDocker(opts) } catch (e) { /* fallback */ }
	}
	return startChild(opts)
}
export { getRunning } from './child-process'
```

## 6. **v2.0 新增**：沙箱工具注入 Pi `src/lib/pi/sandbox-tools.ts`

```tsx
import { defineTool } from '@mariozechner/pi-coding-agent'
import { Type } from '@sinclair/typebox'
import { startSandbox, getRunning } from '@/lib/sandbox'

export function buildSandboxTools(projectId: string, workspaceDir: string) {
	return [
		defineTool({
			name: 'sandbox_run',
			label: '启动沙箱',
			description: '在 workspaceDir 启动项目（默认 pnpm dev），返回 URL 与端口',
			parameters: Type.Object({ command: Type.Optional(Type.String()) }, { additionalProperties: false }),
			execute: async (_id, p: any) => {
				const h = await startSandbox({ projectId, workspaceDir, command: p.command ?? 'pnpm dev' })
				return { content: [{ type: 'text', text: `沙箱已启动：${h.url}` }], details: { url: h.url, port: h.port } }
			},
		}),
		defineTool({
			name: 'sandbox_stop',
			label: '停止沙箱',
			description: '停止当前项目的沙箱进程',
			parameters: Type.Object({}, { additionalProperties: false }),
			execute: async () => {
				const h = getRunning(projectId)
				if (h) await h.stop()
				return { content: [{ type: 'text', text: '已停止' }] }
			},
		}),
		defineTool({
			name: 'sandbox_status',
			label: '沙箱状态',
			description: '返回当前项目沙箱是否运行中',
			parameters: Type.Object({}, { additionalProperties: false }),
			execute: async () => {
				const h = getRunning(projectId)
				return { content: [{ type: 'text', text: h ? `running ${h.url}` : 'idle' }] }
			},
		}),
	]
}
```

在 `dev-agent` 创建 session 时，把 `buildSandboxTools(...)` 合并进 `customTools`，这样 Pi 在生成代码后**可以自主调 `sandbox_run`** 验证。

## 7. 三种用法场景

- **设计模块 UI 原型预览**：把 `design/ui.html` 起一个 `npx serve -l $PORT design/`（或写个 30 行的 mini static server），iframe 嵌入。
- **开发模块 Run**：在 workspaceDir 跑 `pnpm dev` 或 `pnpm start`。
- **审查模块**：直接复用上一个开发模块的 sandbox handle，跑 e2e。

## 8. 安全与限制（v2.0 强化）

- 子进程没有真正隔离，**只用于演示**。
- 不暴露到外网（仅 `localhost`）。
- 子进程单项目 1 个；新启动会复用现有（幂等）。
- 关键 env 不透传：把 `process.env.DEEPSEEK_API_KEY` / `ANTHROPIC_API_KEY` / 其他全部扩展位 Key 从注入 env 中剔除。

```tsx
function sanitizedEnv(port: number) {
	const e = { ...process.env }
	for (const k of ['DEEPSEEK_API_KEY','ANTHROPIC_API_KEY','KIMI_API_KEY','XIAOMI_API_KEY','OPENAI_API_KEY','DASHSCOPE_API_KEY','ZHIPU_API_KEY']) delete e[k]
	return { ...e, PORT: String(port), NODE_ENV: 'production' }
}
```

## 9. 进程清理

- Node 退出钩子：`process.on('SIGINT')` / `'SIGTERM'` → 遍历 registry 全部 stop。
- 端口超 1 小时未访问也自动 stop。

## 10. 可降级清单

- 子进程起不来：API 退化为「展示代码 + 给 zip」模式（`/api/projects/:id/dev/sandbox/run` 返回 `{ url:null, message:'sandbox unavailable, use export zip' }`）。
- Docker 装了但拉镜像失败：自动 fallback 子进程。
- pnpm 慢：可在沙箱启动前在主进程做一次 `pnpm fetch` 缓存。
- Pi 工具调用 `sandbox_run` 超时：在工具内部加 60s 硕果超时，超时返回「未完成启动」但不报错，让 Pi 继续迭代。

### 11.2 §3 子进程沙箱兜底修复

现有 `child-process.ts` 三处必修，否则非 Windows 下一启动就 ENOENT 或 waitPort 超时：

```tsx
// 1. spawn 全平台开 shell（修复 `pnpm exec next start -p $PORT` 复合命令）
const child = spawn(opts.command, [], {
	cwd: opts.workspaceDir,
	env: sanitizedEnv(port),
	stdio: ['ignore', 'pipe', 'pipe'],
	shell: true,   // ← 原仅 isWindows 改为 true
})

// 2. waitPort 30s → 180s（Next.js 冷启动中位数 90s）
await waitPort(port, 180_000)

// 3. $PORT 在传入 spawn 前先字符串展开
const cmd = (cmds.start ?? 'pnpm start').replace(/\$PORT/g, String(port))
```

`startSandbox` 新增 `installFirst: true` 默认值；若 `node_modules` 不存在先同步跑 `pnpm install`（10 min 超时）再 `startChild`：

```tsx
export async function startSandbox(opts: SandboxStartOpts) {
	const exists = getRunning(opts.projectId)
	if (exists) return exists
	const port = acquirePort()
	const cmds = detectStartCommand(opts.workspaceDir)
	if (cmds.install && !fssync.existsSync(path.join(opts.workspaceDir, 'node_modules'))) {
		execSync(cmds.install, { cwd: opts.workspaceDir, stdio: 'inherit', timeout: 10 * 60_000 })
	}
	const start = (cmds.start ?? 'pnpm start').replace(/\$PORT/g, String(port))
	return startChild({ ...opts, command: start, port })
}
```

### 11.3 sanitizedEnv 改白名单 + 按需透传

原黑名单方式（剥离所有 `*_API_KEY`）会顺手把应用本身需要的 LLM Key 也剥掉，应用 500。改为：

- 默认透传 `PATH` / `LANG` / `TZ` / `NODE_ENV` 等基础 env
- 若 workspace 的 `.env.runtime` 显式列出某 Key（如 `DEEPSEEK_API_KEY=true`），从主进程透传该 Key
- 其余敏感 Key 仍剥离

```tsx
function sanitizedEnv(port: number, workspaceDir: string) {
	const base: Record<string, string> = { PATH: process.env.PATH ?? '', LANG: process.env.LANG ?? 'C.UTF-8', TZ: process.env.TZ ?? 'Asia/Shanghai', NODE_ENV: 'production', PORT: String(port) }
	const allowList = parseAllowList(path.join(workspaceDir, '.env.runtime'))
	for (const k of allowList) if (process.env[k]) base[k] = process.env[k]!
	return base
}
```

### 11.7 Patch 模式（J3.3）

`POST /api/projects/:id/dev/patch` 启动 dev-agent 的局部修改模式，system prompt 覆盖为 `DEV_PATCH_SYSTEM`：

```tsx
// src/agents/prompts/dev.ts
export const DEV_PATCH_SYSTEM = `你是开发 Agent 的 patch 模式。任务：仅修改用户指定的文件区间，禁止重写整个项目。
输入：filePath / lineRange / userInstruction。
输出：直接 workspace_write 改写该文件；改完用 workspace_list 确认。
禁止跨文件大范围改动；超出范围请在 PLAN.md 末尾追加 TODO。`
```

与完整 `runDev` 的区别：不走全量代码生成，只做单文件局部修改，适合用户从代码浏览器发起的小范围改动。
